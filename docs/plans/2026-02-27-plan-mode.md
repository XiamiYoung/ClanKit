# Plan Mode Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add automatic plan mode — the agent calls `submit_plan` before executing complex tasks, the UI renders a dark plan card inline in the message bubble, and the user approves/refines/rejects before execution proceeds.

**Architecture:** The agent loop gains a `submit_plan` tool it can self-select. When called, it emits a `plan_submitted` chunk and breaks the loop early. On user approval, ChatsView re-runs the agent with the plan injected into the system prompt. The plan card renders inside the existing assistant message bubble in `ChatWindow.vue` via a new `PlanCard.vue` component.

**Tech Stack:** Electron 31 (CommonJS main process), Vue 3 + Pinia (renderer), IPC via `window.electronAPI.runAgent` / `agent:chunk`

### Step 4: Add `approvePlan`, `rejectPlan`, `refinePlan` functions

Add these three functions near the other agent-related functions:

```js
async function approvePlan(msg) {
  const chatId = chatsStore.activeChatId
  chatsStore.setPlanState(chatId, msg.id, 'approved')

  const savedParams = chatsStore.getPlanRunParams(chatId)
  if (!savedParams) return

  // Format plan as text for system prompt injection
  const injectedPlan = `Title: ${msg.planData.title}\n` +
    msg.planData.steps.map((s, i) => `${i + 1}. ${s.label}`).join('\n')

  // Create a new streaming assistant message for the execute phase
  const newMsgId = uuidv4()
  const chat = chatsStore.chats.find(c => c.id === chatId)
  if (chat?.messages) {
    chat.messages.push({
      id: newMsgId,
      role: 'assistant',
      content: '',
      streaming: true,
      streamingStartedAt: Date.now(),
      segments: []
    })
    chat.isRunning = true
    perChatStreamingSegments.set(chatId, [])
  }

  scrollToBottom()

  try {
    const res = await window.electronAPI.runAgent({ ...savedParams, injectedPlan })

    const finalSegments = perChatStreamingSegments.get(chatId) || []
    const execMsg = chat?.messages?.find(m => m.id === newMsgId)
    if (execMsg) {
      execMsg.streaming = false
      execMsg.segments = finalSegments.map(s => ({ ...s }))
      execMsg.content = finalSegments.filter(s => s.type === 'text').map(s => s.content).join('')
      if (!execMsg.content && res.result) {
        execMsg.segments = [{ type: 'text', content: res.result }]
        execMsg.content = res.result
      }
      if (!res.success) {
        execMsg.segments = [{ type: 'text', content: `Error: ${res.error}` }]
        execMsg.content = `Error: ${res.error}`
      }
      if (execMsg.streamingStartedAt) execMsg.durationMs = Date.now() - execMsg.streamingStartedAt
    }
  } finally {
    if (chat) chat.isRunning = false
    await chatsStore.persist?.()
    scrollToBottom()
  }
}

function rejectPlan(msg) {
  chatsStore.setPlanState(chatsStore.activeChatId, msg.id, 'rejected')
}

function refinePlan(msg) {
  chatsStore.setPlanState(chatsStore.activeChatId, msg.id, 'rejected')
  // Pre-fill the editor — find the RichTextEditor ref and call prefill
  richTextEditorRef.value?.prefill('Refine the plan: ')
}
```

**Note:** `richTextEditorRef` is whatever `ref` ChatsView uses to reference the `RichTextEditor`. Search for `ref="editor"` or similar in ChatsView to find the existing ref name. If ChatsView doesn't hold a direct ref to the editor, check if `ChatWindow` exposes it — you may need to add a `ref` and `defineExpose` call. Use the existing pattern in the codebase.
